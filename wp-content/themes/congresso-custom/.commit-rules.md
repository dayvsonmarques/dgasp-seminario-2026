# Git Commit Rules

## Commit Message Standards

### Rules:
1. **Maximum 12 words**
2. **Language: English**
3. **Prefix required**: feature, hotfix, issue, fix, refactor, docs, style, etc.

### Format:
```
<prefix>: <description in max 12 words>
```

### Examples:
- `feature: add automatic Sass compilation with watch script`
- `fix: resolve CSS cache busting in functions`
- `refactor: convert max-width to min-width media queries`
- `hotfix: correct path navigation in watch script`
- `docs: update Sass compilation guide with examples`
- `style: organize media queries at file bottom`

### Valid Prefixes:
- `feature` - New functionality
- `fix` - Bug fix
- `hotfix` - Urgent production fix
- `refactor` - Code restructuring without behavior change
- `docs` - Documentation changes
- `style` - Code formatting (not CSS styling)
- `test` - Test additions or modifications
- `chore` - Maintenance tasks
- `perf` - Performance improvements
- `issue` - Issue resolution

### Tips:
- Be concise and descriptive
- Use imperative mood ("add" not "added")
- No period at the end
- Focus on what changed, not why
